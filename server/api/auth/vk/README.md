# Привязка и перепривязка VK ID

## Поведение

- **Привязка**: пользователь получает код `PG19-...` (POST `/api/auth/vk/link-code`), отправляет его в сообщество VK; webhook обновляет `User.vk_id`.
- **Перепривязка**: тот же поток. Код выдаётся даже если у пользователя уже есть привязанный VK; после успешной отправки кода новым аккаунтом VK поле `User.vk_id` перезаписывается (отдельная отвязка не нужна).
- Один VK-аккаунт не может быть привязан к разным пользователям: при попытке привязать уже занятый VK другому пользователю операция отклоняется, в VK приходит сообщение об ошибке.

## Чеклист для ручной проверки

1. **Нет VK** — зайти в профиль, нажать «Привязать» VK, получить код, отправить в VK; после привязки в блоке контактов отображается новый VK ID.
2. **Уже есть VK A** — нажать «Перепривязать» VK, получить код без ошибки; отправить код из другого аккаунта VK (B); после успешной обработки в профиле отображается VK B, старый A больше не используется.
3. **VK занят другим пользователем** — пользователь 2 запрашивает код привязки, в VK отправляет сообщение с кодом аккаунт, уже привязанный к пользователю 1; в VK должно прийти сообщение об ошибке, у пользователя 1 привязка не меняется.
